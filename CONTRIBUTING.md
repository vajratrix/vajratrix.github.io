# Contributing to Vajratrix Group

First off, thank you for considering contributing! 🔱 As a non-profit, open-source collective, Vajratrix Group exists because of people like you who are willing to donate time and skill instead of money.

## Ways to Contribute

- **Report bugs** — found something broken? [Open an issue](../../issues/new/choose).
- **Suggest features** — have an idea? We'd love to hear it.
- **Fix issues** — check issues labeled [`good-first-issue`](../../issues?q=is%3Aissue+is%3Aopen+label%3A%22good-first-issue%22) for a good starting point.
- **Improve docs** — even fixing a typo helps.
- **Improve SEO/accessibility/performance** — always welcome on a public-facing site like this.

## Before You Start

1. Check existing [issues](../../issues) and [pull requests](../../pulls) to avoid duplicate work.
2. For anything non-trivial, open an issue first to discuss the change before writing code.
3. Read our [Code of Conduct](CODE_OF_CONDUCT.md) — we keep this a friendly, respectful space.

## Making a Pull Request

1. **Fork** the repository and create your branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes. Keep commits focused and messages clear:
   ```bash
   git commit -m "Fix: broken anchor link in nav"
   ```
3. Test your changes locally by opening `index.html` in a browser (see README for local dev instructions).
4. Push to your fork and open a Pull Request against `main`.
5. Fill out the PR template — describe **what** changed and **why**.
6. One of the maintainers will review and may request changes. Please be patient — this is a volunteer-run project.

## Style Guidelines

- Keep HTML semantic and accessible (proper heading order, `alt` text on images, labeled form fields).
- Match existing code formatting/indentation rather than reformatting unrelated code.
- Avoid adding new dependencies unless discussed in an issue first — we aim to keep this site lightweight and free to host.
- Test that changes don't break mobile responsiveness.

## Reporting Security Issues

Please **do not** open a public issue for security vulnerabilities. Instead, email us directly at **connect@vajratrix.com** with details, and we'll respond as quickly as we can.

## Recognition

All contributors are credited in the repository's contributors list, and significant contributions will be highlighted in our project updates and newsletter.

Thank you for helping Vajratrix Group grow! 🔱
